# 設計概要

## 背景

`snape/RVO2` は ORCA の参照実装として有用ですが、研究室内の学習用途や継続的な保守を考えると、以下の課題があります。

- コードリーディングの前提知識が高い
- 日本語の説明が不足している
- upstream の構成が、そのまま研究室の開発運用に適しているとは限らない

そのため本リポジトリでは、upstream を参照しながら、研究室向けの独自実装を段階的に構築します。

## 基本原則

1. `external/RVO2` は参照専用とし、研究室独自の修正は原則として加えない
2. 実装より先に設計文書を整備する
3. 研究室独自実装は `src/` と `include/` に集約する
4. public API は upstream の概念を踏まえつつ、研究室内で理解しやすい粒度へ再設計する
5. 最終的には upstream 比較で妥当性を検証する

## 実装フェーズ

### Phase 0: 初期構成

- ライセンス表記の整理
- third-party notice の整備
- 設計文書、API 草案、変更記録方針の整備
- ビルド可能な最小スケルトンの作成

### Phase 1: 最小 ORCA 再実装

- 2 次元ベクトル演算
- エージェント状態管理
- 時間更新
- ORCA 制約生成
- 線形計画による速度決定

まずは読みやすさを優先し、必要であれば近傍探索を単純実装から始めます。

### Phase 2: 研究室向け API の具体化

- シナリオ記述
- シミュレーション設定
- エージェントの追加、更新、観測 API
- 比較実験や教材用途を意識したログ/可視化連携点

### Phase 3: upstream 比較と検証

- 単純な群衆シナリオでの挙動比較
- 速度、位置、衝突回避の定性的比較
- 既知ケースに対する回帰テスト追加

## ディレクトリ方針

```text
docs/               設計、API、運用方針
include/cmis/orca/  外部公開ヘッダ
src/                実装本体
examples/           利用例
tests/              自動テスト
external/RVO2/      upstream 参照コード
```

## upstream との関係

- upstream はアルゴリズム理解と比較検証の基準として使う
- 研究室実装は upstream のファイル構成に拘束されない
- upstream 由来コードを持ち込む場合は、出典と改変事実を記録する

## 現時点のスケルトンについて

初期構成で用意する `Simulator` は、将来の ORCA 実装を差し込むための土台です。  
現段階では API とビルド導線の整備を優先しており、完全な衝突回避ロジックはまだ実装していません。
