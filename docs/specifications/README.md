# 仕様書

このディレクトリでは、実装済みの構成、入出力、処理フロー、テスト方針を固定する。

## 基本方針

- 仕様書は「今このリポジトリで何が実装されているか」を記録する
- 設計意図だけでなく、未実装や placeholder も明記する
- 実装を変更したら、同じ change で仕様書も更新する

詳細な運用ルールは [../policies/documentation-sync-policy.md](../policies/documentation-sync-policy.md) を参照してください。
