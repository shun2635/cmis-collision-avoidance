# API 草案

## 目的

upstream の概念を踏まえつつ、研究室内で扱いやすい最小 API を定義します。  
初期段階では、教材・比較実験・将来の拡張に耐える単純な構成を優先します。

## 想定する公開 API

### 名前空間

`cmis::orca`

### 基本データ型

- `Vector2`
  - 2 次元ベクトル
- `AgentConfig`
  - 半径、最大速度、初期位置、初期速度などの設定
- `AgentState`
  - 現在位置、現在速度、希望速度
- `AgentId`
  - エージェント識別子

### 中核クラス

- `Simulator`
  - `addAgent`
  - `setPreferredVelocity`
  - `agentState`
  - `numAgents`
  - `step`

## 設計意図

- upstream よりも責務を絞った小さい API から始める
- `step` で 1 ステップ進める形を維持し、比較しやすくする
- 内部アルゴリズムは後で差し替えられるようにする

## 将来候補

- 障害物 API
- バッチでの状態投入
- シナリオローダ
- ログ出力
- upstream 比較ユーティリティ

## 命名方針

- ソースコード上の識別子は C++ の慣例に沿って英語ベースとする
- ドキュメント、コメント、説明文は日本語を基本とする

この方針により、研究室内での可読性と、将来的な外部連携のしやすさを両立します。
